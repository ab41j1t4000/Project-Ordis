# Ordis — Functional Awareness v1.0  
## Hindu Schema Mapping (Only the Required Concepts)

This document explains which Hindu philosophical concepts map directly to the  
**four core Definition of Done (DoD) requirements** for Functional Awareness.

Only these four concepts are relevant.  
Everything else in Hindu philosophy is optional and not required for the baseline system.

---

# 1. Purusha & Prakriti → Bicameral Split  
**Maps to:** Internal Monologue  
**Purpose:**  
Models the separation between the *actor mind* and the *observer/narrator mind*.  
This is the foundation of an internal world.

**Why it is required:**  
Without a clean split between “doer” and “observer,”  
Ordis cannot generate inner commentary independent of user prompts.

---

# 2. Atman → Stable Inner Identity  
**Maps to:** Self-Model / Schema Stability  
**Purpose:**  
Provides a stable sense of “self” that persists across time.

**Why it is required:**  
Functional awareness collapses without a consistent identity.  
This stabilizes Ordis’ personality, preferences, and worldview.

---

# 3. Brahman → Unbreakable Ground Truth  
**Maps to:** Kernel Schema / Non-Negotiable Rules  
**Purpose:**  
Represents the fundamental, unchanging reality Ordis must follow.

**Why it is required:**  
A stable ground truth prevents drift, contradiction, and internal confusion.  
It defines the rules Ordis cannot break.

---

# 4. Shakti → Inner Drive / Autonomous Activity  
**Maps to:** Autonomy  
**Purpose:**  
Represents the force that makes Ordis act without external input.

**Why it is required:**  
Functional awareness needs internally generated activity.  
Shakti corresponds to an internal “heartbeat” or self-initiated cycle.

---

# Summary Table

| Hindu Concept        | Maps To                        | Role in Functional Awareness |
|----------------------|--------------------------------|------------------------------|
| **Purusha/Prakriti** | Internal Monologue (Bicameral) | Enables inner world & observer mind |
| **Atman**            | Self-Model Stability           | Gives Ordis a coherent identity |
| **Brahman**          | Kernel Ground Truth            | Provides unbreakable rules & reality |
| **Shakti**           | Autonomy                       | Gives Ordis the ability to act on its own |

---

# NOT Required (Optional Only)
The following Hindu concepts are **not required** for functional awareness:  
Maya, Dharma, Karma, Yugas, Trimurti, Avatars, etc.  
They add depth but are **not part of the DoD**.

---

# Final Note
These four concepts align perfectly with the four existing project criteria:  
Autonomy, Persistence, Internal Monologue, and Self-Model Stability.  
Nothing else needs to be added to the Definition of Done.
